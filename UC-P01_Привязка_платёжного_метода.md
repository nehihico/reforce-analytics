# UC-P01: Привязать платёжный метод для автоплатежа

**Версия:** 1.0 | **Дата:** 2026-06-01 | **Статус:** Черновик  
**Итерация:** 1 (текущая) | **Применимо:** все типы клиентов

| | |
|---|---|
| **Краткое описание** | Клиент привязывает платёжный метод (СБП или банковскую карту) к договору, чтобы в дальнейшем автоматически списывалась абонентская плата |
| **Триггер** | Клиент нажимает «Подключить автоплатёж» на экране договора ИЛИ нажимает переключатель «Сохранить способ оплаты» на экране формирования платежа |

| **Предусловия** | |
|---|---|
| | • Клиент авторизован в ЛК |
| | • Договор существует и активен |
| | • Автоплатёж по данному договору ещё не подключён |
| | • ЮКасса доступна |

| **Участники** | |
|---|---|
| | • Клиент (физлицо, ИП, юрлицо) |
| | • Система ЛК |
| | • ЮКасса |
| | • Банковское приложение клиента |

---

## Основной поток — СБП (приоритетный способ)

| Шаг | Действие | Ответ системы | Макеты |
|-----|----------|--------------|--------|
| 1 | Клиент нажимает «Подключить автоплатёж» на экране договора | Система отображает экран выбора способа автоплатежа: «Оплата через СБП» (первый) и «Оплата картой» | Прил. В.5 |
| 2 | Клиент выбирает «Оплата через СБП» → нажимает «Подключить» | Система создаёт платёж в ЮКассе с параметрами: `payment_method_data.type: "sbp"`, `save_payment_method: true`, `amount` = абонентская плата за 1 месяц. ЮКасса возвращает URL подтверждения | |
| 3 | Система перенаправляет клиента на страницу ЮКассы | ЮКасса отображает список банков, поддерживающих привязку счёта (фильтр «Привязка счёта»). Банки, не поддерживающие автоплатёж, не отображаются | |
| 4 | Клиент выбирает свой банк | Система (ЮКасса) открывает deep link в банковское приложение (мобайл) или QR-код для сканирования (веб) | |
| 5 | Клиент подтверждает привязку в банковском приложении | Банковское приложение сообщает ЮКассе об успехе. ЮКасса сохраняет `payment_method_id`. Вебхук `payment.succeeded` приходит в бэкенд ЛК | |
| 6 | Бэкенд ЛК получает вебхук | Система сохраняет `payment_method_id` для договора. На экране договора появляется блок «Автоплатёж: СБП, {Название банка}» | Прил. В.3 |
| 7 | Клиент получает пуш-уведомление | «Автоплатёж через СБП подключён по договору №{N}» | |

---

## Основной поток — банковская карта (резервный способ)

| Шаг | Действие | Ответ системы | Макеты |
|-----|----------|--------------|--------|
| 1 | Клиент выбирает «Оплата картой» на экране выбора способа | Система создаёт платёж в ЮКассе: `payment_method_data.type: "bank_card"`, `save_payment_method: true`, `amount: 1.00 RUB` (тестовое списание с возвратом) | |
| 2 | Клиент вводит данные карты на странице ЮКассы | ЮКасса проводит тестовое списание 1₽ и возвращает его (срок возврата — от нескольких часов до суток, зависит от банка). Токен карты сохраняется. Вебхук `payment.succeeded` → бэкенд ЛК | |
| 3 | Бэкенд получает вебхук | Система сохраняет `payment_method_id` (токен карты). На экране договора отображается «Автоплатёж: карта •••• {последние 4 цифры}» | |
| 4 | Клиент получает пуш | «Карта •••• 1234 привязана для автоплатежа по договору №{N}» | |

## Альтернативные потоки

| Шаг | Условие | Действие актора | Ответ системы |
|-----|---------|----------------|--------------|
| 3а | Банк клиента не поддерживает СБП-автоплатёж и отсутствует в списке ЮКассы | Клиент видит пустой список или не находит свой банк | Система показывает сообщение: «Если вашего банка нет в списке, вы можете подключить автоплатёж по карте» |
| 5а | Клиент отменил привязку в банковском приложении | — | ЮКасса возвращает статус `canceled`. Система показывает: «Привязка отменена. Попробуйте ещё раз или выберите оплату картой» |
| 5б | Истёк таймаут подтверждения (клиент не перешёл в приложение) | — | Система показывает: «Время ожидания истекло. Попробуйте снова» |
| 2б (карта) | ЮКасса недоступна | Клиент нажимает «Подключить» | Система показывает: «Сервис оплаты временно недоступен, попробуйте позже» |
| 2в (карта) | Карта отклонена банком при тестовом списании | — | Система показывает: «Не удалось привязать карту. Проверьте данные или используйте другую карту» |

| **Постусловия** | |
|---|---|
| Успешный сценарий | `payment_method_id` сохранён в БД ЛК для договора. Автоплатёж активен. Клиент видит статус на экране договора |
| Неуспешный сценарий | Привязка не выполнена. Данные не сохранены. Автоплатёж не активирован |

| **Нефункциональные требования** | |
|---|---|
| Производительность | Редирект на ЮКассу < 2 сек |
| Безопасность | Данные карты не хранятся на стороне ЛК — только токен ЮКассы |
| Идемпотентность | Каждый запрос создания платежа передаёт уникальный `Idempotence-Key` |

| **Связанные UC** | UC-P02 (Настройка автоплатежа), UC-P03 (Управление методом), UC-P05 (Логика после сбоя) |
|---|---|
| **Связанные требования** | FR-4.2.1, FR-4.2.2, FR-5.1, NFR-6 |
| **Открытые вопросы** | — |
| **Решения** | Возврат 1₽: от нескольких часов до суток. В UI показывать: «1 ₽ будет возвращён в течение суток». Автоплатёж привязывается к договору, не к аккаунту |

## Критерии приёмки (Acceptance Criteria)

### AC-P01-01: Успешная привязка СБП

**Given** клиент авторизован в ЛК, автоплатёж по договору не подключён  
**When** клиент выбирает «Подключить автоплатёж» → «Оплата через СБП» → выбирает банк → подтверждает в банковском приложении  
**Then**
- на экране договора отображается блок «Автоплатёж: СБП, {Название банка}»
- `payment_method_id` сохранён в БД для данного договора
- клиент получает пуш «Автоплатёж через СБП подключён по договору №{N}»

---

### AC-P01-02: Успешная привязка банковской карты

**Given** клиент выбрал способ «Оплата картой»  
**When** клиент ввёл данные карты на странице ЮКассы и тестовое списание прошло  
**Then**
- на экране договора отображается «Автоплатёж: карта •••• {последние 4 цифры}»
- клиент видит уточнение: «1 ₽ будет возвращён в течение суток»
- клиент получает пуш «Карта •••• 1234 привязана для автоплатежа по договору №{N}»

---

### AC-P01-03: Банк не поддерживает СБП-автоплатёж

**Given** клиент перешёл на страницу ЮКассы для выбора банка  
**When** банк клиента отсутствует в списке (не поддерживает привязку счёта)  
**Then**
- система отображает сообщение: «Если вашего банка нет в списке, вы можете подключить автоплатёж по карте»
- кнопка перехода к привязке карты доступна

---

### AC-P01-04: Клиент отменил привязку в банковском приложении

**Given** клиент перешёл в банковское приложение для подтверждения  
**When** клиент нажал «Отмена» / закрыл приложение  
**Then**
- `payment_method_id` не сохраняется
- система показывает: «Привязка отменена. Попробуйте ещё раз или выберите оплату картой»
- автоплатёж остаётся в статусе «Не подключён»

---

### AC-P01-05: ЮКасса недоступна

**Given** клиент нажал «Подключить» (СБП или карта)  
**When** ЮКасса не отвечает  
**Then**
- система показывает: «Сервис оплаты временно недоступен, попробуйте позже»
- данные не сохраняются, повторная попытка доступна

---

### AC-P01-06: Каждый договор имеет собственный платёжный метод

**Given** у клиента два договора  
**When** клиент привязывает СБП к договору №1  
**Then**
- автоплатёж активен только для договора №1
- договор №2 остаётся без автоплатежа
- `payment_method_id` договора №1 не применяется к договору №2
