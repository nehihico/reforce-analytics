# UC-P03: Управлять платёжным методом автоплатежа

**Версия:** 1.0 | **Дата:** 2026-06-01 | **Статус:** Черновик  
**Итерация:** 1 (текущая) | **Применимо:** все типы клиентов

| | |
|---|---|
| **Краткое описание** | Клиент просматривает текущий способ автоплатежа по договору и меняет его на другой (СБП другого банка, карта, смена с СБП на карту и обратно) |
| **Триггер** | Клиент нажимает «Сменить способ оплаты» в блоке «Автоплатёж» на экране договора |

| **Предусловия** | |
|---|---|
| | • Клиент авторизован в ЛК |
| | • Автоплатёж по договору активирован (есть активный `payment_method_id`) |
| | • ЮКасса доступна |

| **Участники** | |
|---|---|
| | • Клиент |
| | • Система ЛК |
| | • ЮКасса |

---

## Экран управления автоплатежом (отображаемая информация)

Блок «Автоплатёж» на экране договора содержит:
- Текущий способ: СБП ({Название банка}) / карта •••• {последние 4 цифры}
- Дата следующего списания
- Сумма следующего списания
- Кнопка «Сменить способ оплаты»
- Переключатель выключения (OFF)

---

## Основной поток — смена способа оплаты

| Шаг | Действие | Ответ системы | Макеты |
|-----|----------|--------------|--------|
| 1 | Клиент нажимает «Сменить способ оплаты» | Система открывает экран подключения нового способа (идентичен экрану первичного подключения) | Прил. В.5 |
| 2 | Клиент выбирает новый способ (СБП или карта) и проходит привязку | Система выполняет UC-P01 (привязка нового метода) | |
| 3 | Новый метод успешно привязан | Система деактивирует старый `payment_method_id`, сохраняет новый. Блок «Автоплатёж» обновляется | |
| 4 | Клиент получает пуш | «Способ автоплатежа по договору №{N} изменён на {новый способ}» | |

---

## Альтернативные потоки

| Шаг | Условие | Действие актора | Ответ системы |
|-----|---------|----------------|--------------|
| 2а | Клиент закрыл экран привязки, не завершив процесс | Закрывает экран | Старый `payment_method_id` остаётся активным. Автоплатёж продолжает работать со старым методом |
| 2б | Новый СБП-банк не поддерживает автоплатёж | Клиент не находит свой банк в списке | Система показывает: «Если вашего банка нет в списке, подключите автоплатёж по карте» |
| 3а | Привязка нового метода не прошла (ошибка) | — | Старый метод остаётся активным. Система показывает: «Не удалось подключить новый способ оплаты. Попробуйте ещё раз» |

---

## Поток — добавление второго метода (СБП + карта)

| Шаг | Условие | Действие | Результат |
|-----|---------|----------|-----------|
| — | У клиента активен только СБП, он хочет добавить карту как резерв | Клиент нажимает «Добавить резервный способ» | Система привязывает карту как второй `payment_method_id`. При сбое СБП — автоматический фоллбэк на карту |

> Открытый вопрос: явно ли в требованиях предусмотрен отдельный UI для добавления второго (резервного) метода, или это происходит автоматически при наличии двух сохранённых токенов? Требует уточнения.

---

| **Постусловия** | |
|---|---|
| Успешный сценарий | Новый `payment_method_id` активен. Старый деактивирован. Следующее списание пройдёт через новый метод |
| Неуспешный сценарий | Старый метод остаётся активным без изменений |

| **Нефункциональные требования** | |
|---|---|
| Безопасность | Смена метода подтверждается пуш- или SMS-уведомлением |
| Атомарность | Деактивация старого метода происходит только после успешной привязки нового |

| **Связанные UC** | UC-P01 (Привязка метода), UC-P04 (Отмена автоплатежа) |
|---|---|
| **Связанные требования** | FR-4.2.4, FR-5.1 |
| **Решения** | Автоплатёж — на договор. У каждого договора свой `payment_method_id` |
| **Открытые вопросы** | Есть ли отдельный UI для добавления резервного (второго) метода к одному договору? |

## Критерии приёмки (Acceptance Criteria)

### AC-P03-01: Блок «Автоплатёж» отображает актуальный способ

**Given** автоплатёж подключён по договору  
**When** клиент открывает экран договора  
**Then**
- отображается: текущий способ (СБП + название банка / карта •••• 1234), дата следующего списания, сумма
- доступны кнопки «Сменить способ оплаты» и переключатель выключения

---

### AC-P03-02: Успешная смена способа оплаты

**Given** автоплатёж активен, клиент нажал «Сменить способ оплаты»  
**When** клиент прошёл привязку нового метода (UC-P01) успешно  
**Then**
- в блоке «Автоплатёж» отображается новый способ
- старый `payment_method_id` деактивируется только после успешной привязки нового
- клиент получает пуш: «Способ автоплатежа по договору №{N} изменён на {новый способ}»
- следующее списание пройдёт через новый метод

---

### AC-P03-03: Смена прервана — старый метод сохраняется

**Given** клиент открыл экран смены метода  
**When** клиент закрыл экран, не завершив привязку нового метода  
**Then**
- старый `payment_method_id` остаётся активным
- автоплатёж продолжает работать по-прежнему
- никаких уведомлений не отправляется

---

### AC-P03-04: Ошибка привязки нового метода

**Given** клиент пытается сменить способ  
**When** привязка нового метода завершилась ошибкой (банк отклонил, таймаут)  
**Then**
- старый `payment_method_id` остаётся активным (атомарность)
- система показывает: «Не удалось подключить новый способ оплаты. Попробуйте ещё раз»
- блок «Автоплатёж» показывает прежний способ без изменений
