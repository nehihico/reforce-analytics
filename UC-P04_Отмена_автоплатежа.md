# UC-P04: Отменить автоплатёж по договору

**Версия:** 1.0 | **Дата:** 2026-06-01 | **Статус:** Черновик  
**Итерация:** 1 (текущая) | **Применимо:** все типы клиентов

| | |
|---|---|
| **Краткое описание** | Клиент самостоятельно отключает автоплатёж по договору. После отключения списания прекращаются; клиент несёт ответственность за своевременную ручную оплату |
| **Триггер** | Клиент нажимает переключатель «Автоплатёж» (OFF) в блоке «Автоплатёж» на экране договора |

| **Предусловия** | |
|---|---|
| | • Клиент авторизован в ЛК |
| | • Автоплатёж по договору активирован |

| **Участники** | |
|---|---|
| | • Клиент |
| | • Система ЛК |

---

## Основной поток

| Шаг | Действие | Ответ системы | Макеты |
|-----|----------|--------------|--------|
| 1 | Клиент нажимает переключатель «Автоплатёж» (переводит в OFF) | Система отображает диалог подтверждения: «Автоплатёж будет отключён. Не забывайте оплачивать услуги вовремя.» — кнопки «Отключить» и «Отмена» | Прил. В.3 |
| 2 | Клиент нажимает «Отключить» | Система деактивирует `payment_method_id` для договора (помечает как неактивный, не удаляет). Блок «Автоплатёж» переходит в состояние «Не подключён» | |
| 3 | Клиент получает пуш или SMS | «Автоплатёж по договору №{N} отключён. Следите за балансом и оплачивайте вовремя» | |

---

## Альтернативные потоки

| Шаг | Условие | Действие актора | Ответ системы |
|-----|---------|----------------|--------------|
| 1а | Клиент нажал «Отмена» в диалоге подтверждения | Нажимает «Отмена» | Диалог закрывается. Автоплатёж остаётся активным без изменений |
| 2а | Отключение инициируется системой (3 неудачные попытки списания) | — | Система автоматически приостанавливает автоплатёж (см. UC-P02, поток C). Клиент получает пуш с объяснением |

---

## Поведение системы после отключения

| Параметр | Поведение |
|---|---|
| `payment_method_id` | Сохраняется в БД (не удаляется), помечается `is_active: false` |
| Повторное подключение | Клиент может повторно подключить автоплатёж (UC-P01), при этом заново проходит привязку метода |
| Баннер долга | Остаётся на главном экране, если долг не погашен |
| Пуш-напоминания о долге | Продолжают приходить согласно таблице эскалации (FR-4.1.1) |

---

## Сценарий: отключение пуш-напоминаний (не автоплатежа)

> Это отдельный сценарий — клиент может отключить только пуш-напоминания в настройках профиля, не отключая автоплатёж. Баннер долга на главном экране при этом сохраняется.

| Шаг | Действие | Ответ системы |
|-----|----------|--------------|
| 1 | Клиент переходит в «Профиль» → «Настройки уведомлений» | Отображается переключатель «Напоминания о задолженности» |
| 2 | Клиент отключает напоминания | Система прекращает отправку пуш-уведомлений о долге. Баннер долга на главном экране остаётся |

---

| **Постусловия** | |
|---|---|
| Успешный сценарий | Автоплатёж деактивирован. `payment_method_id` сохранён, но не используется. Клиент уведомлён |
| Неуспешный сценарий | Автоплатёж остаётся активным (например, клиент нажал «Отмена» в диалоге) |

| **Нефункциональные требования** | |
|---|---|
| Безопасность | Отключение подтверждается пуш- или SMS-уведомлением |

| **Связанные UC** | UC-P01 (Привязка метода), UC-P03 (Управление методом), UC-P02 (Автоматическое приостановление) |
|---|---|
| **Связанные требования** | FR-4.2.4, FR-4.1.1, NFR-6 |
| **Открытые вопросы** | При повторном подключении автоплатежа — нужно ли заново проходить привязку или система может переиспользовать сохранённый `payment_method_id`? Зависит от срока действия токена в ЮКассе |

## Критерии приёмки (Acceptance Criteria)

### AC-P04-01: Диалог подтверждения перед отключением

**Given** автоплатёж активен по договору  
**When** клиент переводит переключатель «Автоплатёж» в OFF  
**Then**
- система показывает диалог: «Автоплатёж будет отключён. Не забывайте оплачивать услуги вовремя»
- доступны две кнопки: «Отключить» и «Отмена»
- автоплатёж ещё не деактивирован на этом шаге

---

### AC-P04-02: Успешное отключение автоплатежа

**Given** клиент нажал «Отключить» в диалоге подтверждения  
**When** система обрабатывает запрос  
**Then**
- автоплатёж деактивирован: блок «Автоплатёж» переходит в состояние «Не подключён» с кнопкой «Подключить автоплатёж»
- `payment_method_id` сохраняется в БД со статусом `is_active: false` (не удаляется)
- клиент получает пуш: «Автоплатёж по договору №{N} отключён. Следите за балансом и оплачивайте вовремя»
- пуш-напоминания о долге продолжают приходить при наличии задолженности

---

### AC-P04-03: Отмена в диалоге — автоплатёж сохраняется

**Given** клиент открыл диалог подтверждения отключения  
**When** клиент нажимает «Отмена»  
**Then**
- диалог закрывается
- автоплатёж остаётся активным без каких-либо изменений
- никаких уведомлений не отправляется

---

### AC-P04-04: Отключение пуш-напоминаний не отключает автоплатёж

**Given** у клиента активен автоплатёж  
**When** клиент отключает напоминания о задолженности в «Профиль» → «Настройки уведомлений»  
**Then**
- пуш-напоминания о долге прекращаются
- автоплатёж остаётся активным и списывает средства в штатном режиме
- баннер долга на главном экране сохраняется при наличии задолженности

---

### AC-P04-05: Автоматическая приостановка системой (3 неудачные попытки)

**Given** автоплатёж активен, три попытки списания подряд завершились неудачей  
**When** третья попытка отклонена ЮКассой  
**Then**
- автоплатёж переходит в статус `suspended`
- клиент получает пуш: «Автоплатёж по договору №{N} приостановлен: 3 попытки списания не прошли. Оплатите вручную»
- на экране договора отображается статус «Автоплатёж приостановлен» с кнопкой «Возобновить»
