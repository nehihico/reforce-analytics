# UC-P06: Оплатить задолженность — физлицо

**Версия:** 1.0 | **Дата:** 2026-06-02 | **Статус:** Черновик  
**Применимо:** Физлицо  
**Итерация:** 1 (текущая)

| | |
|---|---|
| **Краткое описание** | Клиент-физлицо оплачивает задолженность по договору через экран оплаты: выбирает способ (карта или СБП), вводит сумму и завершает платёж |
| **Триггер** | Клиент нажимает «Оплатить» на экране договора, в баннере долга на главном экране или по deep link из пуш-уведомления |

| | • Тип клиента: физлицо |
| | • По договору есть задолженность |
| | • ЮКасса доступна |

| **Участники** | |
|---|---|

## Итерация 1 — текущая реализация

### Основной поток

| Шаг | Действие | Ответ системы | Макеты |
|-----|----------|--------------|--------|
| 1 | Клиент нажимает «Оплатить» | Открывается bottom sheet «Оплата задолженности»: отображается сумма долга, два блока: «Картой или через СБП» и «По счёту с расчётного счёта» (для ИП, если применимо) | |
| 2 | Клиент выбирает «Оплатить онлайн» (карта или СБП) | Открывается экран «Оплата договора» | |
| 3 | Клиент выбирает привязанную карту или СБП-банк из списка | Система подставляет выбранный метод. Если карт нет — отображается «+ Добавить карту» | |
| 4 | Система предзаполняет сумму и email | В поле «Сумма» — сумма задолженности. В поле «Электронная почта» — email из профиля клиента. Доступны быстрые кнопки: «Всю сумму {X} ₽» и «Оплатить задолженность {Y} ₽» | |
| 5 | Клиент нажимает «Оплатить» | Переход на страницу ЮКассы для подтверждения | |
| 6 | Клиент подтверждает платёж | Вебхук `payment.succeeded` → бэкенд. Баланс договора обновляется немедленно | |
| 7 | Экран успешной оплаты | «Оплата прошла успешно. {сумма} ₽ по договору №{N}». Кнопки: «Подключить автоплатёж» и «Вернуться к договору» | |

---

### Порядок способов оплаты

| Позиция | Способ | Условие отображения |
|---|---|---|
| 1 | СБП (первый) | Всегда |
| 2 | Привязанная карта | Если есть сохранённая карта |
| 3 | + Добавить карту | Всегда |

---

### Предзаполнение суммы

| Ситуация | Что подставляется в поле «Сумма» |
|---|---|
| Переход из баннера долга | Полная сумма задолженности |
| Переход из экрана договора | Полная сумма задолженности |
| Переход из пуша | Сумма долга из пуша |
| Клиент меняет вручную | Любая сумма — поле редактируемо |

---

### Альтернативные потоки

| Шаг | Условие | Действие актора | Ответ системы |
|-----|---------|----------------|--------------|
| 3а | ЮКасса недоступна | Клиент нажимает «Оплатить» | «Сервис оплаты временно недоступен, попробуйте позже» |
| 4а | Клиент хочет сохранить способ для автоплатежа | Устанавливает чекбокс «Сохранить способ оплаты для автоплатежа» (расположен над кнопкой «Оплатить») | При успешной оплате система сохраняет `payment_method_id` и активирует автоплатёж (см. UC-P01) |
| 6а | Платёж отклонён | — | «Платёж не прошёл. Попробуйте другой способ оплаты» |
| 7а | Автоплатёж не подключён, у клиента **нет сохранённых платёжных методов** | Клиент видит экран успеха и нажимает «Оплатить и сохранить карту» | Система открывает экран оплаты договора с предзаполненной суммой задолженности и с активированным чекбоксом «Сохранить способ оплаты для автоплатежа». После успешной оплаты карта сохраняется и автоплатёж активируется |

---

### Экран после успешной оплаты — контекстное предложение автоплатежа

Блок отображается **только если автоплатёж по данному договору не подключён**. Если автоплатёж активен — блок не показывается, экран содержит только кнопку «Вернуться к договору».

Содержимое блока:
- Заголовок: «Подключить автоплатёж, чтобы не забывать»
- Текст: «Автоматическое списание абонентской платы каждый месяц»
- Кнопка «Подключить» (основная) — ведёт на UC-P01
- Кнопка «Вернуться к договору» (вторичная)

|---|---|

| Неуспешный сценарий | Платёж не проведён. Баланс не изменён. Клиент видит ошибку и может повторить |

## Поток: Баннер долга на главном экране

### Описание

Баннер — постоянный элемент главного экрана, отображающийся поверх списка объектов пока по хотя бы одному договору клиента есть задолженность.

### Условия отображения / скрытия

| Условие | Поведение баннера |
|---------|------------------|
| Есть задолженность по 1+ договорам | Баннер отображается |
| Все задолженности погашены | Баннер исчезает (после обновления баланса по вебхуку) |
| Баланс обновился только в 1С, но вебхук ещё не пришёл | Баннер остаётся до получения вебхука `payment.succeeded` |
| Клиент закрыл баннер вручную | Баннер **не закрывается вручную** — исчезает только после погашения долга |

### Содержимое баннера

- Текст: «Задолженность: {сумма} ₽ по {N} договорам» (агрегат по всем должным договорам клиента)
- Кнопка «Оплатить»

### Поведение кнопки «Оплатить»

| Ситуация | Что происходит при нажатии |
|----------|---------------------------|
| Должен **1 договор** | Открывается bottom sheet «Оплата задолженности» для этого договора (основной поток UC-P06, шаг 1) |
| Должны **2+ договора** | Открывается экран «Платежи» с выделенными должными договорами для выбора |

### Критерии приёмки

#### AC-P06-B01: Баннер отображается при наличии долга

**Given** у клиента есть задолженность по одному или нескольким договорам  
**When** клиент открывает главный экран  
**Then**
- баннер отображается с текстом: «Задолженность: {сумма} ₽ по {N} договорам»
- сумма в баннере — агрегат задолженностей по всем должным договорам клиента

---

#### AC-P06-B02: Баннер исчезает после полного погашения долга

**Given** клиент оплатил все задолженности  
**When** бэкенд получил вебхук `payment.succeeded` и обновил баланс  
**Then**
- баннер долга исчезает с главного экрана
- баннер не закрывается вручную — только по факту погашения

---

#### AC-P06-B03: Один должный договор — переход к bottom sheet

**Given** баннер отображается, должен ровно 1 договор  
**When** клиент нажимает «Оплатить» в баннере  
**Then**
- открывается bottom sheet «Оплата задолженности» для этого договора
- в bottom sheet предзаполнена сумма задолженности по договору

---

#### AC-P06-B04: Несколько должных договоров — переход к экрану «Платежи»

**Given** баннер отображается, должны 2+ договора  
**When** клиент нажимает «Оплатить» в баннере  
**Then**
- открывается экран «Платежи»
- должные договоры выделены / отмечены для удобства выбора

## Критерии приёмки (Acceptance Criteria)

### AC-P06-01: Предзаполнение суммы и email

**Given** клиент переходит на экран оплаты из баннера долга или карточки договора  
**When** экран открывается  
**Then**
- поле «Сумма» предзаполнено суммой задолженности
- поле «Электронная почта» предзаполнено email из профиля клиента
- поле email остаётся редактируемым

---

### AC-P06-02: СБП — первый способ в списке

**Given** клиент открыл экран оплаты  
**When** отображается блок «Способ оплаты»  
**Then**
- первым в списке стоит СБП
- следом — привязанные карты (если есть)
- последним — «+ Добавить карту»

---

### AC-P06-03: Чекбокс сохранения способа для автоплатежа

**Given** клиент выбрал способ оплаты и готов платить  
**When** клиент смотрит на экран перед нажатием «Оплатить»  
**Then**
- чекбокс «Сохранить способ оплаты для автоплатежа» расположен над кнопкой «Оплатить» (не под ней)
- при установленном чекбоксе после успешной оплаты активируется автоплатёж

---

### AC-P06-04: Экран успешной оплаты — автоплатёж не подключён

**Given** платёж прошёл успешно, автоплатёж по данному договору **не подключён**  
**When** клиент видит экран подтверждения  
**Then**
- отображается сообщение «Оплата прошла успешно. {сумма} ₽ по договору №{N}»
- ниже — блок «Подключить автоплатёж, чтобы не забывать»
- кнопка «Подключить» ведёт на UC-P01
- кнопка «Вернуться к договору» закрывает экран

---

### AC-P06-04б: Экран успешной оплаты — автоплатёж уже подключён

**Given** платёж прошёл успешно, автоплатёж по данному договору **уже активен**  
**When** клиент видит экран подтверждения  
**Then**
- отображается сообщение «Оплата прошла успешно. {сумма} ₽ по договору №{N}»
- блок «Подключить автоплатёж» **не отображается**
- доступна только кнопка «Вернуться к договору»

---

### AC-P06-04в: Подключение автоплатежа без сохранённой карты — редирект на экран оплаты

**Given** платёж прошёл успешно, автоплатёж не подключён, у клиента **нет сохранённых платёжных методов**  
**When** клиент видит блок «Подключить автоплатёж, чтобы не забывать»  
**Then**
- вместо кнопки «Подключить» отображается кнопка «Оплатить и сохранить карту»
- нажатие открывает экран оплаты договора (UC-P06 основной поток) с предзаполненной суммой задолженности и **чекбоксом «Сохранить способ оплаты для автоплатежа» активированным по умолчанию**
- клиент вводит данные карты или выбирает СБП и оплачивает
- после успешной оплаты: платёжный метод сохраняется, автоплатёж активируется (см. UC-P01)
- клиент видит экран успешной оплаты с подтверждением что автоплатёж подключён

> **Открытый вопрос:** что показывать, если после этой оплаты баланс становится 0 и нет задолженности для предзаполнения суммы? Уточнить на следующей итерации.

---

### AC-P06-05: Успешная оплата обновляет баланс немедленно

**Given** ЮКасса вернула вебхук `payment.succeeded`  
**When** бэкенд обработал вебхук  
**Then**
- баланс договора обновляется немедленно, без ожидания синхронизации с 1С
- баннер долга на главном экране исчезает (если долг погашен полностью)
