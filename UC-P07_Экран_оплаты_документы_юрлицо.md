# UC-P07: Оплатить задолженность и работа с документами — юрлицо и ИП

**Версия:** 1.0 | **Дата:** 2026-06-02 | **Статус:** Черновик  
**Применимо:** Юрлицо, ИП  
**Итерация:** 1 (текущая) + Итерация 2 (следующая)

| | |
|---|---|
| **Краткое описание** | Клиент-юрлицо или ИП оплачивает задолженность по договору: через счёт на оплату (приоритет для юрлица) или онлайн (карта / СБП). Раздел «Отчётные документы» позволяет скачать документы за последние периоды |
| **Триггер** | Клиент нажимает «Оплатить» на карточке договора или переходит по deep link из пуш-уведомления |

| **Предусловия** | |
|---|---|
| | • Клиент авторизован в ЛК |
| | • Тип клиента: юрлицо или ИП (поле `ЮрФизЛицо` из 1С) |
| | • По договору есть задолженность |

| **Участники** | |
|---|---|
| | • Клиент (руководитель юрлица / ИП) |
| | • Бухгалтер (при наличии доступа) |
| | • Система ЛК |
| | • ЮКасса |
| | • 1С (источник данных для документов) |

## Итерация 1 — текущая реализация

---

### Поток 1: Выбор способа оплаты

| Шаг | Действие | Ответ системы |
|-----|----------|--------------|
| 1 | Клиент нажимает «Оплатить» на карточке договора | Открывается bottom sheet «Оплата задолженности» с суммой долга |
| 2 | Bottom sheet отображает два варианта | **Для юрлица:** «Получить счёт» (основная кнопка) и «Оплатить онлайн» (вторичная). **Для ИП:** «Оплатить онлайн» (основная) и «По счёту с расчётного счёта» (вторичная) |
| 3а | Клиент выбирает «Получить счёт» | Переход на экран формирования счёта (поток 2) |
| 3б | Клиент выбирает «Оплатить онлайн» | Переход на экран оплаты картой / СБП (аналогично UC-P06, поток 3) |

---

### Поток 2: Получение счёта на оплату (Итерация 1)

| Шаг | Действие | Ответ системы |
|-----|----------|--------------|
| 1 | Клиент выбрал «Получить счёт» | Система отображает экран с предпросмотром счёта |
| 2 | Экран содержит | Сумма (предзаполнена суммой долга, редактируемая). Реквизиты плательщика (из 1С: название, ИНН/КПП). Реквизиты получателя (из 1С). Поле «Email бухгалтера» (предзаполнено, если указан в профиле) |
| 3 | Клиент нажимает «Скачать счёт (PDF)» | Система генерирует PDF через HTTP-сервис 1С и скачивает его на устройство |
| 4 | Клиент нажимает «Отправить на email» | PDF отправляется на указанный email. Отображается подтверждение: «Счёт отправлен на {email}» |

Счёт действителен 30 дней.

> Кнопки «Сформировать счёт» и «Акт сверки» в разделе документов в Итерации 1 не реализуются. Запланированы в Итерации 2.

---

### Поток 3: Онлайн-оплата (юрлицо / ИП)

Аналогичен UC-P06 (физлицо) со следующим отличием:

| Параметр | Физлицо (UC-P06) | Юрлицо / ИП (UC-P07) |
|---|---|---|
| Приоритет способа | СБП первый | Счёт первый (для юрлица); СБП первый (для ИП) |
| Автоплатёж | Да | Да (доступен, но не приоритетен для юрлица) |
| Документы | Нет | Раздел «Отчётные документы» в карточке договора |

---

### Поток 4: Раздел «Отчётные документы» в карточке договора (Итерация 1)

Раздел отображается только для клиентов типа «Юрлицо» и «ИП». Расположен в карточке договора между блоком баланса и историей операций.

#### Структура раздела

**Последние 3 периода** отображаются непосредственно в разделе:

| Элемент | Описание |
|---|---|
| Строка периода | Название месяца + год, сумма (например, «Апрель 2026, 1 200 ₽») |
| Кнопка «Скачать счёт» | Скачивает PDF счёта за этот период |
| Кнопка «Скачать акт» | Скачивает PDF акта оказанных услуг за этот период |
| Ссылка «Посмотреть все документы» | Открывает отдельную страницу со всеми документами |

Пример отображения:

```
Отчётные документы

Апрель 2026    1 200 ₽    [Скачать счёт]  [Скачать акт]
Март 2026      1 200 ₽    [Скачать счёт]  [Скачать акт]
Февраль 2026   1 200 ₽    [Скачать счёт]  [Скачать акт]

                              Посмотреть все документы →
```

#### Страница «Все документы»

Открывается при нажатии «Посмотреть все документы». Документы сгруппированы по годам.

```
Все документы — Договор №{N}

2026
  Апрель 2026    1 200 ₽    [Скачать акт]  [Скачать счёт]
  Март 2026      1 200 ₽    [Скачать акт]  [Скачать счёт]
  Февраль 2026   1 200 ₽    [Скачать акт]  [Скачать счёт]
  Январь 2026    1 200 ₽    [Скачать акт]  [Скачать счёт]

2025
  Декабрь 2025   1 200 ₽    [Скачать акт]  [Скачать счёт]
  ...
```

Источник данных: документы 1С (через OData или HTTP-сервис).

### Альтернативные потоки

| Шаг | Условие | Действие актора | Ответ системы |
|-----|---------|----------------|--------------|
| 3а | HTTP-сервис 1С недоступен | Клиент нажал «Скачать» | «Документ временно недоступен. Попробуйте позже» |
| 3б | Документ за период отсутствует в 1С | — | Строка периода не отображается |
| 2а | Email бухгалтера не указан в профиле | Клиент нажал «Отправить на email» | Система запрашивает ввод email перед отправкой |

---

| **Постусловия** | |
|---|---|
| Успешный сценарий (счёт) | PDF сформирован и скачан или отправлен на email |
| Успешный сценарий (онлайн) | Платёж проведён. Баланс обновлён |
| Неуспешный сценарий | Клиент видит ошибку с возможностью повторить действие |

| **Нефункциональные требования** | |
|---|---|
| Производительность | Генерация PDF — не более 5 сек |
| Безопасность | Документы доступны только авторизованному клиенту своего договора |
| Доступ бухгалтера | Бухгалтер с доступом к ЛК видит раздел «Отчётные документы» и может инициировать скачивание и оплату |

| **Связанные UC** | UC-P01 (Привязка метода), UC-P06 (Оплата физлицо) |
|---|---|
| **Связанные требования** | FR-4.3.1 (юрлицо), FR-4.3.2, FR-4.3.3, FR-5.3 |
| **Открытые вопросы** | Роль «бухгалтер» для ИП — финальное решение не принято |

## Итерация 2 — следующая (запланировано)

Дополнения к текущей реализации раздела «Отчётные документы»:

| Что добавляется | Описание | Приоритет |
|---|---|---|
| Кнопка «Сформировать счёт» | Генерация счёта на произвольную сумму с выбором периода. Основная красная кнопка в разделе «Документы» | Высокий |
| Кнопка «Акт сверки» | Формирование акта сверки взаиморасчётов за выбранный период (по умолчанию — с начала года). PDF с таблицей начислений и оплат | Высокий |
| Доступ бухгалтера | Руководитель юрлица / ИП приглашает бухгалтера по email. Бухгалтер получает доступ к разделам «Платежи» и «Отчётные документы» | Средний |

### Поток «Сформировать счёт» (Итерация 2)

| Шаг | Действие | Ответ системы |
|-----|----------|--------------|
| 1 | Клиент нажимает «Сформировать счёт» в разделе «Документы» | Открывается форма: поле суммы (предзаполнено суммой долга), реквизиты (авто) |
| 2 | Клиент указывает сумму и нажимает «Сформировать» | Система генерирует PDF |
| 3 | Клиент скачивает или отправляет на email | PDF доступен для скачивания и отправки |

### Поток «Акт сверки» (Итерация 2)

| Шаг | Действие | Ответ системы |
|-----|----------|--------------|
| 1 | Клиент нажимает «Акт сверки» | Открывается выбор периода (по умолчанию: с начала года по текущую дату) |
| 2 | Клиент подтверждает период | Система формирует PDF: шапка с реквизитами сторон, таблица начислений и оплат, сальдо на начало и конец периода |
| 3 | Клиент скачивает или отправляет на email | PDF доступен |

## Критерии приёмки (Acceptance Criteria) — Итерация 1

### AC-P07-01: Адаптация bottom sheet по типу клиента

**Given** клиент нажал «Оплатить» по договору  
**When** открывается bottom sheet  
**Then**
- для юрлица: кнопка «Получить счёт» — основная (красная), «Оплатить онлайн» — вторичная
- для ИП: кнопка «Оплатить онлайн» — основная, «По счёту» — вторичная
- для физлица: bottom sheet не содержит опцию счёта (см. UC-P06)

---

### AC-P07-02: Предзаполнение реквизитов в счёте

**Given** клиент выбрал «Получить счёт»  
**When** открывается экран счёта  
**Then**
- поле «Сумма» предзаполнено суммой задолженности
- реквизиты плательщика (название, ИНН/КПП) подтянуты из 1С
- реквизиты получателя подтянуты из 1С
- поле «Email бухгалтера» предзаполнено, если указан в профиле

---

### AC-P07-03: Раздел «Отчётные документы» — последние 3 периода

**Given** клиент открыл карточку договора (тип: юрлицо или ИП)  
**When** страница загрузилась  
**Then**
- раздел «Отчётные документы» отображается между блоком баланса и историей операций
- показаны последние 3 завершённых периода
- для каждого периода: название месяца + год, сумма, кнопки «Скачать счёт» и «Скачать акт»
- присутствует ссылка «Посмотреть все документы»
- кнопки «Сформировать счёт» и «Акт сверки» в разделе отсутствуют (Итерация 2)

---

### AC-P07-04: Страница всех документов

**Given** клиент нажал «Посмотреть все документы»  
**When** открывается отдельная страница  
**Then**
- документы сгруппированы по годам (новые сверху)
- каждая запись: «{Месяц} {Год}, {сумма} ₽» + кнопки «Скачать акт» и «Скачать счёт»
- документы загружаются из 1С (OData / HTTP-сервис)

---

### AC-P07-05: Недоступность 1С не блокирует весь экран

**Given** HTTP-сервис 1С недоступен  
**When** клиент открывает раздел «Отчётные документы»  
**Then**
- раздел показывает сообщение: «Документы временно недоступны. Попробуйте позже»
- остальные блоки карточки договора (баланс, история операций) отображаются в штатном режиме

---

### AC-P07-06: Физлицо не видит раздел «Отчётные документы»

**Given** клиент авторизован как физлицо  
**When** клиент открывает карточку договора  
**Then**
- раздел «Отчётные документы» не отображается
- экран оплаты не содержит опцию «Получить счёт»
