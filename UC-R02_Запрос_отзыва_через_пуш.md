# UC-R02: Запрос отзыва через пуш-уведомление

**Версия:** 1.2 | **Дата:** 2026-06-03 | **Статус:** Черновик

| | |
|---|---|
| **Краткое описание** | После завершения заказа система отправляет клиенту однократный пуш с предложением оценить комплект. Клиент переходит по пушу, заполняет форму-опросник (3 обязательных оценки по звёздам + опциональный комментарий; при оценке ≤3 по любому вопросу комментарий становится обязательным) и отправляет отзыв на модерацию |
| **Триггер** | Заказ переведён в статус «Завершён» |

| **Предусловия** | |
|---|---|
| | • Клиент авторизован |
| | • Заказ переведён в статус «Завершён» |
| | • Клиент ещё не оставлял отзыв по данному заказу |
| | • Пуш-уведомления разрешены на устройстве клиента |

| **Участники** | |
|---|---|
| | • Клиент |
| | • Система ЛК |

---

## Основной поток

| Шаг | Действие | Ответ системы |
|-----|----------|--------------|
| 1 | Заказ переведён в статус «Завершён» | Система немедленно проверяет: отзыв по заказу не оставлен → отправляет пуш: «Оцените [название комплекта]» |
| 2 | Клиент нажимает на пуш | Приложение открывает bottom sheet с формой-опросником |
| 3 | — | Форма отображает: название комплекта, три оценки по звёздам — «Качество услуги», «Работа менеджера», «Работа монтажника» (1–5, все обязательны), поле для комментария (опционально, до 500 символов), кнопка «Отправить» (неактивна) |
| 4 | Клиент выставляет все три оценки | Звёзды подсвечиваются. Кнопка «Отправить» становится активной после заполнения всех трёх вопросов |
| 5 | Клиент вводит комментарий (опционально) | Счётчик символов отображает остаток (например, «120/500») |
| 6 | Клиент нажимает «Отправить» | Bottom sheet закрывается. Система создаёт отзыв со статусом «На модерации». Отображается экран: «Спасибо! Отзыв появится после проверки» |

---

## Альтернативные потоки

| Условие | Поведение системы |
|---------|------------------|
| Пуш не доставлен / отклонён клиентом | Повторный пуш не отправляется. Отзыв доступен через карточку заказа (UC-R03) |
| Клиент открыл форму, но не отправил | Повторный пуш не отправляется |
| Клиент уже оставил отзыв по этому заказу | Пуш не отправляется |
| Клиент не заполнил все три оценки | Кнопка «Отправить» неактивна, отправка невозможна |
| Любая из оценок ≤3 звёзд | Поле комментария становится обязательным; метка меняется на «Расскажите, что пошло не так *»; кнопка «Отправить» неактивна до заполнения комментария |
| Клиент превысил 500 символов | Ввод ограничен: символы сверх лимита не принимаются |

---

## Постусловие

Отзыв создан со статусом «На модерации».

---

## Критерии приёмки

| # | Критерий | Ожидаемый результат |
|---|---------|-------------------|
| AC-01 | Заказ завершён, отзыв не оставлен, пуши разрешены | Пуш отправлен однократно |
| AC-02 | Заказ завершён, отзыв уже есть | Пуш не отправляется |
| AC-03 | Заказ завершён, пуши отключены | Пуш не отправляется; отзыв доступен в карточке заказа |
| AC-04 | Клиент не нажал пуш | Повторный пуш не отправляется |
| AC-05 | Клиент нажал пуш | Открывается bottom sheet с формой: название комплекта, три вопроса «Качество услуги», «Работа менеджера», «Работа монтажника», поле комментария |
| AC-06 | Клиент заполнил не все три оценки | Кнопка «Отправить» неактивна |
| AC-07 | Клиент заполнил все три оценки (все ≥4) | Кнопка «Отправить» активна; комментарий опционален |
| AC-08 | Любая из оценок ≤3 | Метка комментария меняется на «Расскажите, что пошло не так *»; кнопка неактивна до заполнения комментария |
| AC-09 | Клиент заполнил обязательный комментарий и нажал «Отправить» | Отзыв создан со статусом «На модерации»; bottom sheet закрывается; экран подтверждения отображается |
| AC-10 | Клиент отправил без комментария (все оценки ≥4) | Отзыв создан; поле комментария пустое |
| AC-11 | Клиент вводит комментарий | Счётчик символов работает корректно; ввод ограничен 500 символами |
| AC-12 | Отправка при нестабильном соединении | Отображается сообщение об ошибке; данные не теряются |
