Все переходы

⭕ → 🟢
Клиент привязал СБП или карту через ЮКассу UC-P01
🟢 → ⭕
Клиент нажал «Отключить» и подтвердил в диалоге UC-P04
🟢 → ⏸
Три попытки автосписания подряд не прошли (день 0, +3, +6) UC-P05
🟢 → 🟢
Смена способа оплаты — деактивируется старый, сохраняется новый payment_method_id UC-P03
⏸ → 🟢
Клиент нажал «Возобновить» и прошёл повторную привязку UC-P01
⏸ → ⚫
Клиент отключил автоплатёж вручную пока он приостановлен UC-P04
⚫ → 🟢
Клиент повторно привязал платёжный метод после отключения UC-P01
Норма → P1
В день списания ЮКасса вернула ошибку (недостаток средств, карта заблокирована и т.д.) UC-P02/P05
P1/P2 → Норма
Клиент оплатил вручную или сменил платёжный метод — счётчик попыток сбрасывается UC-P03/P05
Фоллбэк
СБП отклонён → немедленная попытка через карту (если привязаны оба метода). Счётчик не увеличивается UC-P02/P05

Статусы

Не подключён
payment_method_id = null или is_active = false. Начальное состояние.
Активен
is_active = true. Cron списывает в дату начисления. Внутри — три суб-состояния по счётчику попыток.
Приостановлен
status = suspended. Исчерпаны 3 попытки. payment_method_id сохранён, пуши о долге продолжаются.
Отключён
Клиент сам отключил через переключатель. is_active = false, payment_method_id сохранён.

Use Cases в диаграмме

P01
Привязка СБП или карты через ЮКассу. Создаёт активный автоплатёж.
P02
Cron: push за 3 дня → списание в дату начисления → обновление баланса.
P03
Смена метода: атомарно деактивирует старый и сохраняет новый payment_method_id.
P04
Ручное отключение клиентом. Диалог подтверждения → is_active = false.
P05
Сбои: фоллбэк СБП→Карта, повторные попытки +3/+6 дней, приостановка после трёх неудач.

Хранение в БД

is_active
true = активен / false = приостановлен или отключён
status
active · suspended · disabled
retry_count
0→3. Сбрасывается при ручной оплате или смене метода
payment_method_id
Токен ЮКассы. Никогда не удаляется — только деактивируется

Открытые вопросы

Фоллбэк СБП→Карта: в ту же попытку (мгновенно) или через 3 дня? Уточнить с бэкендом.
После приостановки: нужна новая привязка (UC-P01) или можно переиспользовать сохранённый токен?
Технический timeout ЮКассы: считается «официальной» неудачной попыткой или нет?
